# Replox — AI 聊天回复助手

一款专为 macOS 设计的轻量级通用屏幕自动化工具。它通过**屏幕图像比对**和**原生 OCR 文本分析**，自动监控您指定的任意应用窗口区域。当检测到新内容时，自动调用 AI 大模型生成智能回复并模拟键鼠输入发送。

> ⚠️ **用户须知**：本工具是通用的屏幕自动化框架，用户需自行确保使用场景符合目标应用的服务条款和相关法律法规。

---

## ✨ 核心特性

1. **通用应用支持**：可用于任意桌面应用的屏幕自动化，用户自行框选目标区域。
2. **交互式区域选取**：启动时通过直观的半透明置顶遮罩层，用鼠标直接框选导航列表、内容区和输入框。
3. **免侵入设计**：通过纯视觉截图 + 键鼠模拟实现，不注入任何目标应用进程。
4. **macOS 原生 OCR**：使用 `ocrmac` 调用苹果官方 Vision Kit 框架，极速且高精度识别中英文文本。
5. **智能自适应延时**：活跃时高频巡检（默认 5 秒），无变化时逐步退避延长检查时间（最高 60 秒）。
6. **剪贴板安全输入**：自动复制并使用 Cmd+V 粘贴消息，规避中文输入法带来的输入问题。
7. **AI 智能回复**：预配置通义千问大模型生成个性化回复，API 不可用时支持本地关键字库自动降级。

## 🛠️ 安装与配置

### 1. 创建虚拟环境并安装依赖：
```bash
uv venv
source .venv/bin/activate
uv pip install -r requirements.txt
```

### 2. 授予 macOS 系统权限（⚠️ 关键步骤）
由于此工具涉及全屏截图和鼠标键盘模拟，您**必须**在 macOS 系统设置中开启对应权限：
1. **屏幕录制 (Screen Recording)**：系统设置 > 隐私与安全性 > 屏幕录制
2. **辅助功能 (Accessibility)**：系统设置 > 隐私与安全性 > 辅助功能

### 3. 配置 AI API
在 `config.json` 中填入您的 AI 模型 API 凭证：
```json
{
    "api_key": "您的 API Key",
    "workspace_id": "您的工作空间 ID"
}
```

---

## 🚀 运行指南

### 启动程序
```bash
.venv/bin/python main.py
```

程序启动后，通过 GUI 引导完成以下配置：
1. **框选导航列表区域**（目标应用的联系人/消息列表）
2. **框选内容展示区域**（消息/内容滚动区域）
3. **框选输入框区域**（文本输入区域）

框选完成后，坐标自动保存到 `config.json`，下次运行无需重复配置。

### 使用自动发送模式
```bash
.venv/bin/python main.py --auto-reply
```

## ⚙️ 配置说明 (`config.json`)

| 配置项 | 说明 | 默认值 |
|---|---|---|
| `api_key` | AI 模型 API Key | 空 |
| `workspace_id` | AI 工作空间 ID | 空 |
| `model` | AI 模型名称 | `qwen-plus` |
| `target_app_name` | 目标应用名称（留空则不检测前台） | 空 |
| `base_interval` | 基本检查间隔（秒） | `5.0` |
| `max_interval` | 最长检查间隔（秒） | `60.0` |
| `badge_color_r_min` | 通知标记颜色 R 通道最小值 | `210` |
| `badge_color_g_max` | 通知标记颜色 G 通道最大值 | `110` |
| `badge_color_b_max` | 通知标记颜色 B 通道最大值 | `110` |
| `system_prompt` | AI 回复的行为规则 Prompt | 内置默认 |
| `persona` | AI 角色人设 | 空 |
| `knowledge_base` | 知识库参考资料 | 空 |

## 📂 项目结构

* `main.py`：主控制循环与多窗口切换调度。
* `region_selector.py`：全屏半透明选区界面。
* `screenshot_engine.py`：基于 `mss` 的极速屏幕捕获。
* `change_detector.py`：通过图像比对判定内容区域是否发生变更。
* `ocr_engine.py`：macOS 原生 Apple Vision OCR 文本解析。
* `auto_replier.py`：AI 大模型接口调用和本地兜底关键字匹配。
* `chat_controller.py`：使用 `pyautogui` 和剪贴板驱动目标应用的输入操作。
